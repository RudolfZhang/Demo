The Protected Users group has its own ACL

The Protected Users group is a default group designed to effectively protect and manage credentials within the enterprise. Members of this group have non-configurable protections applied to their accounts. Membership in the Protected Users group is meant to be restrictive and proactively secure.

For example, if you add Jeff Ford to the Account Operators group, his account becomes protected. The Help Desk, which can reset all other user passwords in the Employees OU, cannot reset Jeff's password.

  •   

    By default, the protected groups include Domain Admins, Enterprise Admins, Administrators, and Account Operators. Members of a protected group receive a copy of an ACL from the protected group.

    To view the membership of the Protected Users group you must enable Advanced Features (View menu) in the Active Directory Users and Computers tool.
